# Amazon S3

> Object storage service and the data foundation of virtually all AWS architectures.
> [[Lambda]] can be triggered by S3 events.
> [[EventBridge]] can match S3 event patterns.
> [[CloudWatch]] provides S3 storage and request metrics.
> [[CloudTrail]] records S3 API calls (Data Events).
> Metaflow uses S3 as its artifact store; see [[Metaflow工作流框架]].

> 对象存储服务，几乎所有 AWS 架构的数据基石。
> [[Lambda]] 可由 S3 事件触发。
> [[EventBridge]] 可匹配 S3 事件模式。
> [[CloudWatch]] 提供 S3 存储和请求指标。
> [[CloudTrail]] 记录 S3 API 调用（Data Events）。
> Metaflow 以 S3 为 artifact store，见 [[Metaflow工作流框架]]。

## Core Concepts / 核心概念

| Concept / 概念 | Description / 说明 |
|------|------|
| **Bucket** | Container for objects, globally unique name / 对象的容器，全球唯一命名 |
| **Object** | Basic unit of storage (Key + Value + Metadata) / 存储的基本单位（Key + Value + Metadata） |
| **Key** | Unique identifier within a Bucket (full path, e.g. `data/2026/file.csv`) / 对象在 Bucket 中的唯一标识（完整路径） |
| **Prefix** | Key prefix, simulates directory structure (S3 has no real directories) / Key 的前缀，模拟目录结构（S3 本身没有目录概念） |
| **Region** | AWS Region where the Bucket resides / Bucket 所在的 AWS 区域 |
| **ACL** | Access Control List (deprecated, use Bucket Policy instead) / 访问控制列表（已不推荐，用 Bucket Policy 替代） |
| **ETag** | Object hash for integrity verification / 对象的哈希值，用于验证完整性 |

### Limits / 限制

| Item / 项目 | Limit / 限制 |
|------|------|
| Max single object / 单对象最大 | 5 TB |
| Max single PUT / 单次 PUT 最大 | 5 GB (use Multipart Upload for larger) / 5 GB（超过用 Multipart Upload） |
| Bucket count / Bucket 数量 | 100 / account (can be raised to 1000) / 100 / 账户（可提升到 1000） |
| Object count / 对象数量 | Unlimited / 无限制 |
| Max Key length / Key 最大长度 | 1,024 bytes / 1,024 字节 |

## Storage Classes / 存储类别

| Class / 类别 | Use Case / 适用场景 | Min Storage / 最低存储期 | Retrieval Fee / 检索费用 | Per GB/month / 每 GB/月 |
|------|---------|----------|---------|---------|
| **S3 Standard** | Frequent access / 频繁访问 | None / 无 | None / 无 | ~$0.023 |
| **S3 Intelligent-Tiering** | Uncertain access patterns / 访问模式不确定 | None / 无 | None / 无 | ~$0.023 (+ monitoring fee / + 监控费) |
| **S3 Standard-IA** | Infrequent access (monthly) / 不频繁访问（月级） | 30 days / 30 天 | Yes / 有 | ~$0.0125 |
| **S3 One Zone-IA** | Infrequent + reproducible data / 不频繁 + 可重建数据 | 30 days / 30 天 | Yes / 有 | ~$0.01 |
| **S3 Glacier IR** | Archive (ms retrieval) / 归档（毫秒检索） | 90 days / 90 天 | Yes / 有 | ~$0.004 |
| **S3 Glacier Flexible** | Archive (min-to-hour retrieval) / 归档（分钟到小时检索） | 90 days / 90 天 | Yes / 有 | ~$0.0036 |
| **S3 Glacier Deep Archive** | Long-term archive (12h retrieval) / 长期归档（12 小时检索） | 180 days / 180 天 | Yes / 有 | ~$0.00099 |

### Intelligent-Tiering Tiers / Intelligent-Tiering 层级

```
Frequent Access tier (< 30 days without access)
频繁访问层（< 30 天无访问）
    ↓ automatic / 自动
Infrequent Access tier (30-90 days without access)
不频繁访问层（30-90 天无访问）
    ↓ automatic / 自动
Archive Instant Access tier (90-180 days without access)
归档即时访问层（90-180 天无访问）
    ↓ opt-in / 可选开启
Archive Access tier (> 90 days)
归档访问层（> 90 天）
    ↓ opt-in / 可选开启
Deep Archive tier (> 180 days)
深度归档层（> 180 天）
```

## Basic Operations / 操作基础

### Create Bucket / 创建 Bucket

```bash
aws s3api create-bucket \
    --bucket my-data-bucket \
    --region us-east-1

# Non us-east-1 requires LocationConstraint
# 非 us-east-1 需要指定 LocationConstraint
aws s3api create-bucket \
    --bucket my-data-bucket \
    --region ap-northeast-1 \
    --create-bucket-configuration LocationConstraint=ap-northeast-1
```

### Upload and Download / 上传和下载

```bash
# Single file / 单文件
aws s3 cp local-file.csv s3://my-bucket/data/file.csv
aws s3 cp s3://my-bucket/data/file.csv local-file.csv

# Directory sync / 目录同步
aws s3 sync ./local-dir/ s3://my-bucket/prefix/ --delete
aws s3 sync s3://my-bucket/prefix/ ./local-dir/

# Recursive copy / 递归复制
aws s3 cp s3://bucket-a/prefix/ s3://bucket-b/prefix/ --recursive
```

### Python SDK

```python
import boto3

s3 = boto3.client('s3')

# Upload / 上传
s3.upload_file('local.csv', 'my-bucket', 'data/file.csv')

# Download / 下载
s3.download_file('my-bucket', 'data/file.csv', 'local.csv')

# Read into memory / 读取到内存
response = s3.get_object(Bucket='my-bucket', Key='data/file.json')
content = response['Body'].read().decode('utf-8')

# Write string/bytes / 写入字符串/字节
s3.put_object(
    Bucket='my-bucket',
    Key='data/output.json',
    Body=json.dumps(data).encode('utf-8'),
    ContentType='application/json'
)

# List objects / 列出对象
paginator = s3.get_paginator('list_objects_v2')
for page in paginator.paginate(Bucket='my-bucket', Prefix='data/'):
    for obj in page.get('Contents', []):
        print(obj['Key'], obj['Size'])
```

### Presigned URL / 预签名 URL

```python
# Generate temporary download link (accessible without AWS credentials)
# 生成临时下载链接（无需 AWS 凭证即可访问）
url = s3.generate_presigned_url(
    'get_object',
    Params={'Bucket': 'my-bucket', 'Key': 'data/file.csv'},
    ExpiresIn=3600  # 1 hour valid / 1 小时有效
)

# Generate temporary upload link / 生成临时上传链接
url = s3.generate_presigned_url(
    'put_object',
    Params={'Bucket': 'my-bucket', 'Key': 'uploads/user-file.csv'},
    ExpiresIn=3600
)
```

### Multipart Upload

Large files (> 100 MB) should use multipart upload.

> 大文件（> 100 MB）应使用分段上传。

```python
from boto3.s3.transfer import TransferConfig

config = TransferConfig(
    multipart_threshold=100 * 1024 * 1024,  # 100 MB
    multipart_chunksize=100 * 1024 * 1024,
    max_concurrency=10
)

s3.upload_file('large-file.tar.gz', 'my-bucket', 'backups/large.tar.gz',
               Config=config)
```

## Permissions & Access Control / 权限与访问控制

### Permission Evaluation Order / 权限评估顺序

```
Explicit Deny → Bucket Policy → IAM Policy → Default Deny
显式 Deny → Bucket Policy → IAM Policy → 默认 Deny
```

### Bucket Policy

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AllowReadFromVPC",
      "Effect": "Allow",
      "Principal": "*",
      "Action": ["s3:GetObject"],
      "Resource": "arn:aws:s3:::my-bucket/*",
      "Condition": {
        "StringEquals": {
          "aws:sourceVpce": "vpce-xxxxxxxx"
        }
      }
    },
    {
      "Sid": "DenyUnencryptedUploads",
      "Effect": "Deny",
      "Principal": "*",
      "Action": "s3:PutObject",
      "Resource": "arn:aws:s3:::my-bucket/*",
      "Condition": {
        "StringNotEquals": {
          "s3:x-amz-server-side-encryption": "aws:kms"
        }
      }
    },
    {
      "Sid": "DenyInsecureTransport",
      "Effect": "Deny",
      "Principal": "*",
      "Action": "s3:*",
      "Resource": [
        "arn:aws:s3:::my-bucket",
        "arn:aws:s3:::my-bucket/*"
      ],
      "Condition": {
        "Bool": {"aws:SecureTransport": "false"}

```

### Block Public Access (must enable) / Block Public Access（必须开启）

```bash
# Account-level block all public access
# 账户级别阻断所有公开访问
aws s3control put-public-access-block \
    --account-id <account-id> \
    --public-access-block-configuration \
        BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true

# Bucket-level / Bucket 级别
aws s3api put-public-access-block \
    --bucket my-bucket \
    --public-access-block-configuration \
        BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true
```

### S3 Access Points

Create independent access entries for different applications/teams to simplify permission management.

> 为不同应用/团队创建独立的访问入口，简化权限管理。

```bash
aws s3control create-access-point \
    --account-id <account-id> \
    --name analytics-ap \
    --bucket my-data-bucket \
    --vpc-configuration VpcId=vpc-xxxxxxxx

# Access via Access Point / 通过 Access Point 访问
aws s3 cp s3://arn:aws:s3:us-east-1:<account-id>:accesspoint/analytics-ap/data/file.csv .
```

## Encryption / 加密

| Method / 方式 | Description / 说明 | Key Management / 密钥管理 |
|------|------|---------|
| **SSE-S3** | S3-managed keys (default) / S3 托管密钥（默认） | AWS managed / AWS 管理 |
| **SSE-KMS** | KMS keys / KMS 密钥 | Custom CMK available, with audit log / 可用自定义 CMK，有审计日志 |
| **SSE-C** | Customer-provided keys / 客户提供密钥 | Customer managed / 客户管理 |
| **Client-side encryption** | Encrypt before upload / 上传前加密 | Fully customer managed / 完全客户管理 |

```bash
# Set default encryption to SSE-KMS / 设置默认加密为 SSE-KMS
aws s3api put-bucket-encryption \
    --bucket my-bucket \
    --server-side-encryption-configuration '{
        "Rules": [{
            "ApplyServerSideEncryptionByDefault": {
                "SSEAlgorithm": "aws:kms",
                "KMSMasterKeyID": "arn:aws:kms:us-east-1:<account-id>:key/xxx"
            },
            "BucketKeyEnabled": true
        }]
    }'
```

**Bucket Key**: Reduces KMS API calls and lowers encryption costs. Recommended to always enable with SSE-KMS.

> **Bucket Key**：减少 KMS API 调用，降低加密成本。建议 SSE-KMS 时始终开启。

## Versioning / 版本控制

```bash
# Enable versioning / 开启版本控制
aws s3api put-bucket-versioning \
    --bucket my-bucket \
    --versioning-configuration Status=Enabled

# List all versions of an object / 列出对象的所有版本
aws s3api list-object-versions --bucket my-bucket --prefix data/file.csv

# Restore old version (copy old version as current) / 恢复旧版本（复制旧版本为当前版本）
aws s3api copy-object \
    --bucket my-bucket \
    --copy-source "my-bucket/data/file.csv?versionId=OLD_VERSION_ID" \
    --key data/file.csv

# Permanently delete a specific version / 永久删除特定版本
aws s3api delete-object --bucket my-bucket --key data/file.csv --version-id VERSION_ID
```

### MFA Delete

Requires MFA to delete object versions or suspend versioning.

> 需要 MFA 才能删除对象版本或暂停版本控制。

```bash
aws s3api put-bucket-versioning \
    --bucket my-bucket \
    --versioning-configuration Status=Enabled,MFADelete=Enabled \
    --mfa "arn:aws:iam::<account-id>:mfa/root-device 123456"
```

## Performance Optimization / 性能优化

### Request Performance / 请求性能

S3 supports per-prefix:

> S3 单 Prefix 支持：

- **GET**: 5,500 requests/sec / 5,500 请求/秒
- **PUT/COPY/POST/DELETE**: 3,500 requests/sec / 3,500 请求/秒

Multiple prefixes scale linearly.

> 多 Prefix 可以线性扩展。

### Optimization Strategies / 优化手段

| Scenario / 场景 | Optimization / 优化方法 |
|------|---------|
| Many small file reads / 大量小文件读取 | Distribute across multiple prefixes / 分散到多个 Prefix |
| Large file upload / 大文件上传 | Multipart Upload + parallelism / Multipart Upload + 并行 |
| Slow list operations / 列表操作慢 | Use S3 Inventory instead of ListObjects / 用 S3 Inventory 替代 ListObjects |
| Cross-region access / 跨区域访问 | S3 Transfer Acceleration |
| High-frequency access hotspots / 高频访问热点 | CloudFront CDN caching / CloudFront CDN 缓存 |
| Batch operations / 批量操作 | S3 Batch Operations |

### S3 Select / Glacier Select

Query directly on S3 to reduce data transfer.

> 在 S3 端直接查询，减少数据传输。

```python
# Query only needed columns and rows from CSV
# 从 CSV 中只查询需要的列和行
response = s3.select_object_content(
    Bucket='my-bucket',
    Key='data/large.csv',
    ExpressionType='SQL',
    Expression="SELECT s.name, s.amount FROM s3object s WHERE s.amount > '100'",
    InputSerialization={'CSV': {'FileHeaderInfo': 'USE'}},
    OutputSerialization={'JSON': {}}
)

for event in response['Payload']:
    if 'Records' in event:
        print(event['Records']['Payload'].decode())
```

## Data Transfer / 数据传输

### Cross-Region Replication (CRR) / 跨区域复制（CRR）

```bash
# Prerequisite: both source and destination Buckets must have versioning enabled
# 前提：源和目标 Bucket 都开启版本控制
aws s3api put-bucket-replication \
    --bucket source-bucket \
    --replication-configuration '{
        "Role": "arn:aws:iam::<account-id>:role/s3-replication-role",
        "Rules": [{
            "Status": "Enabled",
            "Priority": 1,
            "Filter": {"Prefix": "critical/"},
            "Destination": {
                "Bucket": "arn:aws:s3:::dest-bucket-us-west-2",
                "StorageClass": "STANDARD_IA"
            },
            "DeleteMarkerReplication": {"Status": "Enabled"}
        }]
    }'
```

### Transfer Acceleration

```bash
# Enable acceleration (uses CloudFront edge nodes)
# 开启加速（使用 CloudFront 边缘节点）

    --bucket my-bucket \
    --accelerate-configuration Status=Enabled

# Use accelerated endpoint / 使用加速端点

    --endpoint-url https://my-bucket.s3-accelerate.amazonaws.com
```

## Cost Optimization / 成本优化

### Pricing Highlights (us-east-1) / 定价要点（us-east-1）

| Item / 项目 | Standard | Notes / 说明 |
|------|---------|------|
| Storage / 存储 | $0.023/GB/month | |
| PUT/COPY/POST | $0.005/1000 requests / 次 | |
| GET/SELECT | $0.0004/1000 requests / 次 | |
| Data out (to internet) / 数据传出（到互联网） | $0.09/GB | Free between same-region AWS services / 同区域 AWS 服务间免费 |
| Data in / 数据传入 | Free / 免费 | |

### Optimization Strategies / 优化策略

1. **Lifecycle policies / 生命周期策略**: Automatically transition old data to cold storage / 自动将旧数据转冷存储
2. **Intelligent-Tiering**: Use when access patterns are uncertain / 访问模式不确定时使用
3. **Clean up incomplete Multipart Uploads / 清理未完成的 Multipart Upload**: Lifecycle rule `AbortIncompleteMultipartUpload`
4. **Delete old versions / 删除旧版本**: Clean up after enabling versioning / 版本控制开启后注意清理
5. **S3 Analytics**: Analyze access patterns to decide storage class transitions / 分析访问模式，决定是否转存储类别
6. **VPC Endpoint (Gateway)**: Free, avoids NAT Gateway traffic costs / 免费，避免 NAT Gateway 流量费
7. **Same-region deployment / 同区域部署**: S3 and EC2/Lambda in same region, free transfer / S3 与 EC2/Lambda 同区域，传输免费
8. **Compression / 压缩**: Compress before storing (gzip, zstd) / 存储前压缩（gzip、zstd）

### S3 Inventory

Periodically generate a bucket content manifest, replacing expensive ListObjects calls.

> 定期生成 Bucket 内容清单，替代昂贵的 ListObjects。

```bash
aws s3api put-bucket-inventory-configuration \
    --bucket my-bucket \
    --id weekly-inventory \
    --inventory-configuration '{
        "Destination": {
            "S3BucketDestination": {
                "Bucket": "arn:aws:s3:::my-inventory-bucket",
                "Format": "CSV",
                "Prefix": "inventory"
            }
        },
        "IsEnabled": true,
        "Id": "weekly-inventory",
        "IncludedObjectVersions": "Current",
        "OptionalFields": ["Size", "LastModifiedDate", "StorageClass", "ETag"],
        "Schedule": {"Frequency": "Weekly"}
    }'
```
