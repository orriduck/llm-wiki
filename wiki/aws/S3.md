> [[EventBridge]] 可匹配 S3 事件模式。
> [[CloudWatch]] 提供 S3 存储和请求指标。
> [[CloudTrail]] 记录 S3 API 调用（Data Events）。
> Metaflow 以 S3 为 artifact store，见 [[Metaflow工作流框架]]。

## 核心概念

| 概念 | 说明 |
|------|------|
| **Bucket** | 对象的容器，全球唯一命名 |
| **Object** | 存储的基本单位（Key + Value + Metadata） |
| **Key** | 对象在 Bucket 中的唯一标识（完整路径，如 `data/2026/file.csv`） |
| **Prefix** | Key 的前缀，模拟目录结构（S3 本身没有目录概念） |
| **Region** | Bucket 所在的 AWS 区域 |
| **ACL** | 访问控制列表（已不推荐，用 Bucket Policy 替代） |
| **ETag** | 对象的哈希值，用于验证完整性 |

### 限制

| 项目 | 限制 |
|------|------|
| 单对象最大 | 5 TB |
| 单次 PUT 最大 | 5 GB（超过用 Multipart Upload） |
| Bucket 数量 | 100 / 账户（可提升到 1000） |
| 对象数量 | 无限制 |
| Key 最大长度 | 1,024 字节 |

## 存储类别

| 类别 | 适用场景 | 最低存储期 | 检索费用 | 每 GB/月 |
|------|---------|----------|---------|---------|
| **S3 Standard** | 频繁访问 | 无 | 无 | ~$0.023 |
| **S3 Intelligent-Tiering** | 访问模式不确定 | 无 | 无 | ~$0.023（+ 监控费） |
| **S3 Standard-IA** | 不频繁访问（月级） | 30 天 | 有 | ~$0.0125 |
| **S3 One Zone-IA** | 不频繁 + 可重建数据 | 30 天 | 有 | ~$0.01 |
| **S3 Glacier IR** | 归档（毫秒检索） | 90 天 | 有 | ~$0.004 |
| **S3 Glacier Flexible** | 归档（分钟到小时检索） | 90 天 | 有 | ~$0.0036 |
| **S3 Glacier Deep Archive** | 长期归档（12 小时检索） | 180 天 | 有 | ~$0.00099 |

### Intelligent-Tiering 层级

```
频繁访问层（< 30 天无访问）
    ↓ 自动
不频繁访问层（30-90 天无访问）
    ↓ 自动
归档即时访问层（90-180 天无访问）
    ↓ 可选开启
归档访问层（> 90 天）
    ↓ 可选开启
深度归档层（> 180 天）
```

## 操作基础

### 创建 Bucket

```bash
aws s3api create-bucket \
    --bucket my-data-bucket \
    --region us-east-1

# 非 us-east-1 需要指定 LocationConstraint
aws s3api create-bucket \
    --bucket my-data-bucket \
    --region ap-northeast-1 \
    --create-bucket-configuration LocationConstraint=ap-northeast-1
```

### 上传和下载

```bash
# 单文件
aws s3 cp local-file.csv s3://my-bucket/data/file.csv
aws s3 cp s3://my-bucket/data/file.csv local-file.csv

# 目录同步
aws s3 sync ./local-dir/ s3://my-bucket/prefix/ --delete
aws s3 sync s3://my-bucket/prefix/ ./local-dir/

# 递归复制
aws s3 cp s3://bucket-a/prefix/ s3://bucket-b/prefix/ --recursive
```

### Python SDK

```python
import boto3

s3 = boto3.client('s3')

# 上传
s3.upload_file('local.csv', 'my-bucket', 'data/file.csv')

# 下载
s3.download_file('my-bucket', 'data/file.csv', 'local.csv')

# 读取到内存
response = s3.get_object(Bucket='my-bucket', Key='data/file.json')
content = response['Body'].read().decode('utf-8')

# 写入字符串/字节
s3.put_object(
    Bucket='my-bucket',
    Key='data/output.json',
    Body=json.dumps(data).encode('utf-8'),
    ContentType='application/json'
)

# 列出对象
paginator = s3.get_paginator('list_objects_v2')
for page in paginator.paginate(Bucket='my-bucket', Prefix='data/'):
    for obj in page.get('Contents', []):
        print(obj['Key'], obj['Size'])
```

### Presigned URL（预签名 URL）

```python
# 生成临时下载链接（无需 AWS 凭证即可访问）
url = s3.generate_presigned_url(
    'get_object',
    Params={'Bucket': 'my-bucket', 'Key': 'data/file.csv'},
    ExpiresIn=3600  # 1 小时有效
)

# 生成临时上传链接
url = s3.generate_presigned_url(
    'put_object',
    Params={'Bucket': 'my-bucket', 'Key': 'uploads/user-file.csv'},
    ExpiresIn=3600
)
```

### Multipart Upload

大文件（> 100 MB）应使用分段上传：

```python
from boto3.s3.transfer import TransferConfig

config = TransferConfig(
    multipart_threshold=100 * 1024 * 1024,  # 100 MB
    multipart_chunksize=100 * 1024 * 1024,
    max_concurrency=10
)

s3.upload_file('large-file.tar.gz', 'my-bucket', 'backups/large.tar.gz',
               Config=config)
```

## 权限与访问控制

### 权限评估顺序

```
显式 Deny → Bucket Policy → IAM Policy → 默认 Deny
```

### Bucket Policy

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AllowReadFromVPC",
      "Effect": "Allow",
      "Principal": "*",
      "Action": ["s3:GetObject"],
      "Resource": "arn:aws:s3:::my-bucket/*",
      "Condition": {
        "StringEquals": {
          "aws:sourceVpce": "vpce-xxxxxxxx"
        }
      }
    },
    {
      "Sid": "DenyUnencryptedUploads",
      "Effect": "Deny",
      "Principal": "*",
      "Action": "s3:PutObject",
      "Resource": "arn:aws:s3:::my-bucket/*",
      "Condition": {
        "StringNotEquals": {
          "s3:x-amz-server-side-encryption": "aws:kms"
        }
      }
    },
    {
      "Sid": "DenyInsecureTransport",
      "Effect": "Deny",
      "Principal": "*",
      "Action": "s3:*",
      "Resource": [
        "arn:aws:s3:::my-bucket",
        "arn:aws:s3:::my-bucket/*"
      ],
      "Condition": {
        "Bool": {"aws:SecureTransport": "false"}
      }
    }
  ]
}
```

### Block Public Access（必须开启）

```bash
# 账户级别阻断所有公开访问
aws s3control put-public-access-block \
    --account-id 123456789 \
    --public-access-block-configuration \
        BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true

# Bucket 级别
aws s3api put-public-access-block \
    --bucket my-bucket \
    --public-access-block-configuration \
        BlockPublicAcls=true,IgnorePublicAcls=true,BlockPublicPolicy=true,RestrictPublicBuckets=true
```

### S3 Access Points

为不同应用/团队创建独立的访问入口，简化权限管理：

```bash
aws s3control create-access-point \
    --account-id 123456789 \
    --name analytics-ap \
    --bucket my-data-bucket \
    --vpc-configuration VpcId=vpc-xxxxxxxx

# 通过 Access Point 访问
aws s3 cp s3://arn:aws:s3:us-east-1:123456789:accesspoint/analytics-ap/data/file.csv .
```

## 加密

| 方式 | 说明 | 密钥管理 |
|------|------|---------|
| **SSE-S3** | S3 托管密钥（默认） | AWS 管理 |
| **SSE-KMS** | KMS 密钥 | 可用自定义 CMK，有审计日志 |
| **SSE-C** | 客户提供密钥 | 客户管理 |
| **客户端加密** | 上传前加密 | 完全客户管理 |

```bash
# 设置默认加密为 SSE-KMS
aws s3api put-bucket-encryption \
    --bucket my-bucket \
    --server-side-encryption-configuration '{
        "Rules": [{
            "ApplyServerSideEncryptionByDefault": {
                "SSEAlgorithm": "aws:kms",
                "KMSMasterKeyID": "arn:aws:kms:us-east-1:123456789:key/xxx"
            },
            "BucketKeyEnabled": true
        }]
    }'
```

> **Bucket Key**：减少 KMS API 调用，降低加密成本。建议 SSE-KMS 时始终开启。

## 版本控制

```bash
# 开启版本控制
aws s3api put-bucket-versioning \
    --bucket my-bucket \
    --versioning-configuration Status=Enabled

# 列出对象的所有版本
aws s3api list-object-versions --bucket my-bucket --prefix data/file.csv

# 恢复旧版本（复制旧版本为当前版本）
aws s3api copy-object \
    --bucket my-bucket \
    --copy-source "my-bucket/data/file.csv?versionId=OLD_VERSION_ID" \
    --key data/file.csv

# 永久删除特定版本
aws s3api delete-object --bucket my-bucket --key data/file.csv --version-id VERSION_ID
```

### MFA Delete

需要 MFA 才能删除对象版本或暂停版本控制：

```bash
aws s3api put-bucket-versioning \
    --bucket my-bucket \
    --versioning-configuration Status=Enabled,MFADelete=Enabled \
    --mfa "arn:aws:iam::123456789:mfa/root-device 123456"
```

## 事件通知

### 方式一：S3 Event Notifications（传统）

```bash
aws s3api put-bucket-notification-configuration \
    --bucket my-bucket \
    --notification-configuration '{
        "LambdaFunctionConfigurations": [{
            "LambdaFunctionArn": "arn:aws:lambda:us-east-1:123456789:function:process-upload",
            "Events": ["s3:ObjectCreated:*"],
            "Filter": {
                "Key": {
                    "FilterRules": [
                        {"Name": "prefix", "Value": "uploads/"},
                        {"Name": "suffix", "Value": ".csv"}
                    ]
                }
            }
        }]
    }'
```

### 方式二：EventBridge（推荐）

```bash
# 开启 S3 → EventBridge 通知
aws s3api put-bucket-notification-configuration \
    --bucket my-bucket \
    --notification-configuration '{
        "EventBridgeConfiguration": {}
    }'

# 在 EventBridge 中匹配 S3 事件
# 见 [[EventBridge#事件模式匹配]]
```

> **EventBridge 优势**：更灵活的过滤、多目标、Archive/Replay、跨账户。

## 性能优化

### 请求性能

S3 单 Prefix 支持：
- **GET**: 5,500 请求/秒
- **PUT/COPY/POST/DELETE**: 3,500 请求/秒

多 Prefix 可以线性扩展。

### 优化手段

| 场景 | 优化方法 |
|------|---------|
| 大量小文件读取 | 分散到多个 Prefix |
| 大文件上传 | Multipart Upload + 并行 |
| 列表操作慢 | 用 S3 Inventory 替代 ListObjects |
| 跨区域访问 | S3 Transfer Acceleration |
| 高频访问热点 | CloudFront CDN 缓存 |
| 批量操作 | S3 Batch Operations |

### S3 Select / Glacier Select

在 S3 端直接查询，减少数据传输：

```python
# 从 CSV 中只查询需要的列和行
response = s3.select_object_content(
    Bucket='my-bucket',
    Key='data/large.csv',
    ExpressionType='SQL',
    Expression="SELECT s.name, s.amount FROM s3object s WHERE s.amount > '100'",
    InputSerialization={'CSV': {'FileHeaderInfo': 'USE'}},
    OutputSerialization={'JSON': {}}
)

for event in response['Payload']:
    if 'Records' in event:
        print(event['Records']['Payload'].decode())
```

## 数据传输

### 跨区域复制（CRR）

```bash
# 前提：源和目标 Bucket 都开启版本控制
aws s3api put-bucket-replication \
    --bucket source-bucket \
    --replication-configuration '{
        "Role": "arn:aws:iam::123456789:role/s3-replication-role",
        "Rules": [{
            "Status": "Enabled",
            "Priority": 1,
            "Filter": {"Prefix": "critical/"},
            "Destination": {
                "Bucket": "arn:aws:s3:::dest-bucket-us-west-2",
                "StorageClass": "STANDARD_IA"
            },
            "DeleteMarkerReplication": {"Status": "Enabled"}
        }]
    }'
```

### Transfer Acceleration

```bash
# 开启加速（使用 CloudFront 边缘节点）
aws s3api put-bucket-accelerate-configuration \
    --bucket my-bucket \
    --accelerate-configuration Status=Enabled

# 使用加速端点
aws s3 cp large-file.tar.gz s3://my-bucket/file.tar.gz \
    --endpoint-url https://my-bucket.s3-accelerate.amazonaws.com
```

## 成本优化

### 定价要点（us-east-1）

| 项目 | Standard | 说明 |
|------|---------|------|
| 存储 | $0.023/GB/月 | |
| PUT/COPY/POST | $0.005/1000 次 | |
| GET/SELECT | $0.0004/1000 次 | |
| 数据传出（到互联网） | $0.09/GB | 同区域 AWS 服务间免费 |
| 数据传入 | 免费 | |

### 优化策略

1. **生命周期策略**：自动将旧数据转冷存储
2. **Intelligent-Tiering**：访问模式不确定时使用
3. **清理未完成的 Multipart Upload**：生命周期规则 `AbortIncompleteMultipartUpload`
4. **删除旧版本**：版本控制开启后注意清理
5. **S3 Analytics**：分析访问模式，决定是否转存储类别
6. **VPC Endpoint (Gateway)**：免费，避免 NAT Gateway 流量费
7. **同区域部署**：S3 与 EC2/Lambda 同区域，传输免费
8. **压缩**：存储前压缩（gzip、zstd）

### S3 Inventory

定期生成 Bucket 内容清单，替代昂贵的 ListObjects：

```bash
aws s3api put-bucket-inventory-configuration \
    --bucket my-bucket \
    --id weekly-inventory \
    --inventory-configuration '{
        "Destination": {
            "S3BucketDestination": {
                "Bucket": "arn:aws:s3:::my-inventory-bucket",
                "Format": "CSV",
                "Prefix": "inventory"
            }
        },
        "IsEnabled": true,
        "Id": "weekly-inventory",
        "IncludedObjectVersions": "Current",
        "OptionalFields": ["Size", "LastModifiedDate", "StorageClass", "ETag"],
        "Schedule": {"Frequency": "Weekly"}
    }'
```

## CLI 速查

```bash
# Bucket 操作
aws s3 ls
aws s3 mb s3://BUCKET
aws s3 rb s3://BUCKET --force

# 对象操作
aws s3 ls s3://BUCKET/PREFIX/
aws s3 cp FILE s3://BUCKET/KEY
aws s3 cp s3://BUCKET/KEY FILE
aws s3 mv s3://BUCKET/KEY1 s3://BUCKET/KEY2
aws s3 rm s3://BUCKET/KEY
aws s3 rm s3://BUCKET/PREFIX/ --recursive

# 同步
aws s3 sync LOCAL s3://BUCKET/PREFIX/ --delete --exclude "*.tmp"
aws s3 sync s3://BUCKET/PREFIX/ LOCAL

# 高级
aws s3api head-object --bucket BUCKET --key KEY  # 查看元数据
aws s3api put-object-tagging --bucket BUCKET --key KEY \
    --tagging 'TagSet=[{Key=env,Value=prod}]'

# 大小统计
aws s3 ls s3://BUCKET --recursive --summarize --human-readable
```
